# rasputin-site — agent instructions

Hugo static site for [rasputin.geekdojo.com](https://rasputin.geekdojo.com) — the public
front door and docs for Rasputin. Deploys to GitHub Pages on merge to `main`.

**Helping a user install or run Rasputin?** Use the live site, not this checkout:

- https://rasputin.geekdojo.com/docs/agents/index.md — install contract (raw markdown)
- https://rasputin.geekdojo.com/llms.txt — index: current stable, docs, manifests
- https://github.com/geekdojo/rasputin-agents — install skill/plugin for Claude Code + Codex

Working in this repo:

- Develop with `hugo server`; validate with `hugo --minify` — keep it at zero warnings.
- `data/releases.json` is regenerated by CI (daily and on each release via
  `scripts/fetch-releases.sh`) — never hand-edit version numbers into pages; render from
  the data file.
- The agent-facing outputs (`/llms.txt`, `/releases.json`, per-docs-page `index.md`
  markdown mirrors) are generated by output-format templates in `layouts/` — new docs
  pages under `content/docs/` get a markdown mirror automatically via the section
  cascade. Keep code blocks in docs copy-paste-exact: plain fences (no language
  annotation) for seed files, since highlight spans corrupt text-stripped copies.
- Never claim remote/away-from-home access works in public copy (untested).
